3) MUST-HAVE FEATURES (SCOPE)
============================================================

A) Domain model:
- Product: name, slug, defaultMaxActivations, createdAt
- Customer: email, name, createdAt
- License:
  - key (human-friendly), status (active/suspended/expired/revoked)
  - type (trial/monthly/yearly/lifetime)
  - expiresAt nullable (null = lifetime)
  - maxActivations nullable (fallback to product default)
  - productId, customerId
  - createdAt, updatedAt
- Machine:
  - licenseId, fingerprint (hashed string), activatedAt, lastSeenAt
  - revokedAt nullable
- ActivationLog:
  - licenseId, machineId nullable (when blocked), ip, userAgent
  - eventType (activate/validate/deactivate/blocked)
  - reason nullable, createdAt

B) Public API (apps/server):
Implement routes (REST) OR via oRPC if repo prefers; keep consistent with existing server patterns:
- POST /v1/licenses/activate
- POST /v1/licenses/validate
- POST /v1/licenses/deactivate

Request (activate/validate):
{
  "licenseKey": "XXXX-XXXX-XXXX",
  "productSlug": "my-product",
  "machineId": "HASHED_HWID"
}

Rules:
- License must exist, match product, status active, not expired
- When activating:
  - If machine fingerprint already exists and not revoked => OK (idempotent)
  - Else count active machines (revokedAt is null) for license
  - Compare to maxActivations (license.maxActivations ?? product.defaultMaxActivations)
  - If over limit => reject with reason "MAX_ACTIVATIONS_REACHED"
- When validating:
  - Do not create machine automatically unless you decide to treat validate as "heartbeat".
  - Update lastSeenAt if machine exists.
- When deactivating:
  - Mark machine revokedAt, log event.

Responses:
- Always return a stable schema:
Success:
{
  "ok": true,
  "license": {
    "status": "active",
    "type": "monthly",
    "expiresAt": "2026-01-01T00:00:00.000Z",
    "maxActivations": 3,
    "productSlug": "my-product"
  },
  "machine": {
    "fingerprint": "HASHED_HWID",
    "revokedAt": null
  }
}
Failure:
{
  "ok": false,
  "error": {
    "code": "LICENSE_NOT_FOUND | PRODUCT_MISMATCH | LICENSE_REVOKED | LICENSE_SUSPENDED | LICENSE_EXPIRED | MAX_ACTIVATIONS_REACHED | MACHINE_NOT_FOUND",
    "message": "human-readable",
    "details": { ...optional }
  }
}
